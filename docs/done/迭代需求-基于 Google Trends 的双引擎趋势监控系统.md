基于 Google Trends 的双引擎趋势监控系统
一、 核心商业意图 (Core Intent)
在瞬息万变的全球 AI 市场，SEO 的核心壁垒在于**“时间差”与“资源分配”**。本系统的核心意图分为两翼：
1. 抢新词（探雷引擎）： 在最新的开源模型或 AI 工具从极客圈向大众圈蔓延的“破零”瞬间，第一时间拉响警报，抢占 Google 收录的黄金 24 小时（0 到 1）。
2. 盯老词（资产引擎）： 监控已沉淀一定流量基础的 AI 概念，通过分析其相对市场份额的增减，决定是增加高质量外链投入，还是及时止损撤出资源（1 到 100）。
二、 底层测量原理 (Underlying Principle)
痛点： Google Trends API 返回的 0-100 数据是动态归一化的相对值，受样本量和时间窗口影响极大，极易产生“虚假繁荣”的尖峰，且无法横向比较不同词汇的真实体量。
解法（联合请求与相对锚定）： 强制在同一次 API 请求中，将目标词与极度稳定的“锚点词 (Anchor Word)”绑定。通过计算目标词与锚点词的比值，将无意义的相对指数还原为具有商业指导价值的**“真实市场渗透率”**。
三、 锚点词选择策略
最终意图：“找机会、上站、冲 Google TOP10”——SEO 转化逻辑 
**“意图驱动型锚点策略 (Intent-Driven Anchor Strategy)”**的三大铁律：
铁律一：锚点即“品类霸主” (抢夺核心搜索意图)
你选择的锚点词，必须是那个赛道里目前占据最大自然搜索流量、且心智最成熟的 C 端/B 端产品。
- 逻辑： 找 Midjourney V8、Runway Gen-4.5、Eleven v3、Suno v5。
- 意图匹配： 搜这些霸主词的用户，就是你最渴望的精准目标客户。把新词和霸主词绑在一起测算，一旦新词起量，你就能立刻利用霸主的巨大搜索基数，通过发布“替代品 (Alternatives)”或“硬核对比 (X vs Y)”文章，实现精准的流量截胡。
铁律二：严格“赛道隔离” (确保内容逻辑自洽)
苹果只能和苹果比，绝不能跨界。你的数据库必须按生成类型（如生图、生视频、生音频 等）进行强分类，每个分类死死绑定其专属的霸主锚点。
- 逻辑：
  - Image 新词 ➡️ 强制绑定 Midjourney V8
  - Video 新词 ➡️ 强制绑定 Runway Gen-4.5
  - Audio 新词 ➡️ 拆分为 Suno v5 (音乐) 和 Eleven v3 (人声)
- 意图匹配： 只有同赛道对比，你的数据才能直接翻译成 Google 喜欢的语义连贯的文章标题。你不能写 “Vidu Q3 vs 传统修图软件”，但你可以理直气壮地写 “Vidu Q3: The Runway Gen-4 Killer?”，后者的点击率和搜索排名潜力是前者的十倍。
铁律三：阈值直接映射“建站动作” (数据指导执行)
算出来的相对比例（新词/锚点词），不再是为了画一张好看的趋势图，而是触发你立刻去上站、做 SEO 的“发令枪”。

四、 核心数据预计算 (Data Preparation)
获取到包含 [目标词 K, 锚点词 A] 的时间序列数据后，首先进行降噪与指标计算：
1. 真实比例转换 (Ratio Series): 将每日相对值转化为穿透率。
2. $$R_i = \frac{K_i}{A_i + 0.01}$$
3. 消除周末噪音 (SMA): 计算 7 天与 30 天简单移动平均线，提取核心趋势。
4. $$SMA_7 = \frac{1}{7} \sum_{i=n-6}^{n} R_i$$
5. $$SMA_{30} = \frac{1}{30} \sum_{i=n-29}^{n} R_i$$
6. 短期爆发速度 (Velocity): 衡量最近 48 小时的异动倍数。
7. $$Velocity = \frac{Avg(R_{n}, R_{n-1})}{Avg(R_{n-2} \to R_{n-6}) + 0.01}$$

六、 判定算法实现树 (Decision Algorithm)
在后端执行时，按照以下严格的优先级顺序进行 IF/ELSE 判定过滤：
1. 执行优先级 1：捕捉急速破圈 (绑定 T1 锚点)
  - IF $SMA_{30} < 0.05$ (历史基数极低)
  - AND $Velocity > 2.5$ (近两日热度膨胀 2.5 倍以上)
  - AND $R_n > 0.05$ (最新穿透率达到最低建站阈值)
  - THEN 输出标签：🚀 急速破圈
2. 执行优先级 2：捕捉老词翻红 (绑定 T2 锚点)
  - IF $SMA_7 > SMA_{30} \times 1.25$ (短期均线高出长期均线 25% 以上，形成金叉)
  - AND $SMA_{30} > 0.1$ (存在真实的长期流量底盘)
  - THEN 输出标签：🔄 老词翻红
3. 执行优先级 3：捕捉热度衰退 (绑定 T2 锚点)
  - IF $SMA_7 < SMA_{30} \times 0.75$ (短期热度较月度均值暴跌 25% 以上)
  - AND $Velocity < 1.0$ (无企稳反弹迹象)
  - THEN 输出标签：📉 热度衰退
4. 执行优先级 4：确立稳健增长 (绑定 T2 锚点)
  - IF 不满足上述条件
  - AND $SMA_7 \approx SMA_{30}$ (允许 $\pm 20\%$ 的正常波动)
  - AND $SMA_{30} > 0.1$ (流量底盘稳固)
  - THEN 输出标签：📈 稳健增长
