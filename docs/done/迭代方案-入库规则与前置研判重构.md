# 迭代方案：入库规则与前置研判重构 (V4 极客封口版)

本方案在 V3 前期防线的基础上，针对边界工程漏洞做了极其硬核的修补。补齐了全局 IO 互斥锁的恐慌解锁（panic unlock）安全机制，规范了存量清理的工作流路径（扫描飞书数据回填缓存），为这套“查杀+防御”架构打上了最后一块无懈可击的安全补丁。

---

## 一、方案核心演进要点 (V4 终审项)
1. **死锁免疫（Deadlock Immunity）**：`write_keyword.py` 中的写入动作将 `fcntl.LOCK_UN` 放入了 `finally` 块中。如果大批量并发写入时某一条发生 JSON 解析级宕机，保证锁必然被丢弃，全盘服务永不挂死。
2. **闭源自查收割链（Auto-Generate）**：弃用简单的 JSON 文件接收口，将逻辑升级为自动化连携，从现有的状态网路（Feishu API 或 内部缓存）抽检废词（满足 >30d 且无 Trends 热度趋势的存量旧数据），一键自动批量埋回 Graveyard 黑名单中永世不得翻身。

#!/usr/bin/env python3
import json, time, os, datetime, fcntl

GRAVEYARD_FILE = '/root/.openclaw/workspace/data/seo-radar-graveyard.json'
STATE_FILE = '/root/.openclaw/workspace/data/seo-radar-state.json'

def auto_compact():
    """压缩瘦身，定期释放超过 90 天以上的旧黑名单解禁并缩小文件存储"""
    if not os.path.exists(GRAVEYARD_FILE): return
    with open(GRAVEYARD_FILE, 'a+') as f:
        fcntl.flock(f, fcntl.LOCK_EX)
        try:
            f.seek(0)
            content = f.read()
            if not content: return
            gw_data = json.loads(content)
            now = datetime.datetime.now()
            new_data = {kw: meta for kw, meta in gw_data.items() 
                        if meta.get('timestamp') and (now - datetime.datetime.strptime(meta['timestamp'], "%Y-%m-%d %H:%M:%S")).days < 90}
            f.seek(0)
            f.truncate()
            json.dump(new_data, f, ensure_ascii=False, indent=2)
            print(f"磁盘压缩结束：原留存 {len(gw_data)} 条，去沉疴现保留 {len(new_data)} 条生存。")
        except json.JSONDecodeError: pass
        finally:
            fcntl.flock(f, fcntl.LOCK_UN)

def _bulk_migrate(dirty_list):
    """隐式私有批量导库锁服务，避免代码重复"""
    with open(GRAVEYARD_FILE, 'a+') as f:
