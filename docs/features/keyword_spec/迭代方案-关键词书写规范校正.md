# 迭代方案：关键词书写规范校正 (v1.1)

基于全面的独立代码审查与需求对齐，此方案确立为最终可执行版本。

## 零、核心结论总结

1. **方向完全正确**：官方名确权是 SEO 基础，不仅解决 Trends 分数稀释，更能避免同义多词带来的权重内耗。
2. **Phase 1（归一化统一）是地基**：目前 `state_io.normalize` 与 `dedup_check.normalize` 的撕裂已造成 state库 内部的大量碎片化，也是导致去重防线偶发失效和 `write_keyword.py` 重复入库的罪魁祸首。
3. **Phase 2（web_search 确权）是核心**：利用 LLM 现有的 web_search 能力做官方名字段提取，成本为零，收效显著。
4. **Phase B（搭便车）仅作辅助**：部分源（如 GitHub/HF）名称提取过于生硬，不作为主力，返回 null 更稳妥。
