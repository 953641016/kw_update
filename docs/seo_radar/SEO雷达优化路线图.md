# SEO雷达模块迭代路线图

本路线图旨在指引 SEO雷达模块 的后续迭代与优化方向。主要目标是提升数据防泄漏及健壮度，深化去重及竞品识别逻辑，并逐步推进全栈自动化建站联动。当前系统架构已基本成型，下面列出潜在迭代阶段：

### 阶段一：稳定性升维与防抖 (Short-term)
1. **抓取降级与错误恢复机制强化**
   - 痛点：X(Twitter) 及 新闻聚合偶发 429 或限流拦截。
   - 目标：丰富抓取类的 Fallback 手段，结合现有 `web_search` 逻辑优化智能降级算法；完善 API 超时与空数据重试。
2. **LLM 耗时优化与指令微调**
   - 痛点：任务过重的 `RULES.md` 和并发推断可能导致单次会话超时或幻觉处理。
   - 目标：拆解巨石 Prompt，让信息提取、规则过滤、及外部工具调用实现串行分离（Agent Pipeline 层面的优化）。

### 阶段二：数据精密清洗与知识挖掘 (Mid-term)
1. **多级去重引擎（语义级）强化**
   - 痛点：现有的 `dedup_check.py` 以词汇为核心，可能无法识别异形同义词（如 Luma Uni-1 与 Luma Uni 1.0 的漏网状况）。
   - 目标：对去重脚本中引入基于 embedding 或本地小轻量 LLM 的相似度判别，构建强大的实体统一规范流。
2. **动态 SERP 判断字典化**
   - 痛点：针对 `serp_query.py` 中何为官网、何为纯竞品的辨析高度依赖黑名单。
   - 目标：自动累积竞品白名单与黑名单，甚至可自动定期反查哪些竞品 SaaS 已经死亡，提升机会计算可靠性。

### 阶段三：长尾扩张与全自动发版 (Long-term)
1. **长尾生态簇群深挖**
   - 目标：将抓取出的主力 `product` 词作为种子节点，延伸自动派生其对应的功能/场景流词汇 (`demand`词)，一发双爆，形成树状结构推至飞书库。
2. **内容资产转化打通 (TODO 推进)**
   - 目标：接通代码中的 `文案生成规格` TODO，SEO 雷达推送结束后，如果是高质量的 🔴高优先 关键词，通过无缝衔接自动触发「文案 Agent」构建页面并上站。
